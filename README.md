# NEREL_MultiTask_model
Многоцелевая модель для одновременного извлечения именованных сущностей (NER) и классификации событий/отношений в текстах на русском языке.

## Описание проекта

В проекте реализована **многоцелевая модель на базе трансформеров**, которая одновременно решает две задачи обработки текста:

- **Named Entity Recognition (NER)** — извлечение именованных сущностей
- **Классификация событий** — многометочная классификация типов событий в тексте

Модель обучается на датасете **NEREL** — корпусе русскоязычных текстов с разметкой сущностей и событий.

### Ссылка на датасет: https://huggingface.co/datasets/iluvvatar/NEREL

Основная идея проекта — показать, как **multi-task обучение** позволяет одной модели решать несколько NLP-задач, используя общий encoder.

---

# Задачи

## 1. Named Entity Recognition (NER)

Задача извлечения сущностей на уровне токенов.

Используется схема BIO-разметки. Модель предсказывает метку для каждого токена текста.

---

## 2. Классификация событий

Задача **многометочной классификации**.

Для каждого текста модель предсказывает, какие **типы событий** присутствуют в документе.

Один текст может содержать несколько событий.

---

# Этапы проекта

## 1. Разведочный анализ данных (EDA)

Проведен анализ структуры датасета:

- частоты типов сущностей
- частоты типов событий
- распределение длины текстов
- количество сущностей на документ

Это позволяет понять:

- дисбаланс классов
- сложность задачи
- особенности корпуса.

---

## 2. Предобработка данных

Реализован пайплайн обработки данных:

- загрузка JSONL файлов
- парсинг сущностей
- парсинг событий
- построение BIO-разметки
- формирование multi-label вектора событий


---

## 3. Токенизация

Используется **HuggingFace AutoTokenizer**.

Особенность задачи — выравнивание меток сущностей с subword-токенами.

Реализовано:

- сопоставление BIO-меток с токенами
- игнорирование дополнительных subword-токенов
- padding и attention mask

---

# Архитектура модели

Используется **совместная архитектура (Joint Model)**.
Transformer Encoder (BERT)
│
├── Head для NER
│
└── Head для классификации событий


Encoder извлекает общее представление текста, которое используется для обеих задач.

---

# Обучение модели

Обучение реализовано на **PyTorch**.

Основные этапы:

1. токенизация текста
2. формирование батчей
3. прямой проход модели
4. вычисление функций потерь
5. обратное распространение ошибки
6. обновление весов

Выполнено 2 варианта кастомной функции потерь - простая сумма (по умолчанию) и uncertainty weighting. Переключение между лоссами осуществляется через флаг use_uncertainty_weight.

# Оптимизация модели

После обучения проведена оптимизация инференса:

- **динамическая квантизация модели (torch.quantization.quantize_dynamic)**
- измерение скорости работы

**Эффект квантизации:**
*   Ускорение инференса в **1.35 раза**.
*   Уменьшение размера модели в **1.56 раза** (с 711 МБ до 455 МБ).
*   Минимальная потеря качества (менее 3% по NER F1).
